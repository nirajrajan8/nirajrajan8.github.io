Appium 2.x

The modern, recommended way to manage the Appium server lifecycle in code.

Manual Server Startup
// Requires the Appium server to be running manually
// from the command line: `appium`
// No code example here as it's not a programmatic step.
Programmatic Server Management
// Example for starting and stopping the service
AppiumDriverLocalService service = new AppiumServiceBuilder()
    .withIPAddress("127.0.0.1")
    .usingPort(4723)
    .build();

service.start();

// ... run tests here ...

Core Capabilities and Options

Configuring the driver for different platforms (Android and iOS).

Android Options (UiAutomator2)

Appium 2.x

The modern way to set desired capabilities for Android using the UI Automator 2 driver.

Setting Android Options
// Requires: import io.appium.java_client.android.options.UiAutomator2Options;
UiAutomator2Options options = new UiAutomator2Options();
options.setDeviceName("emulator-5554")
    .setPlatformName("Android")
    .setAutomationName("UiAutomator2")
    .setAppPackage("com.android.calculator2")
    .setAppActivity(".Calculator");

// Create the driver with the options
AndroidDriver driver = new AndroidDriver(new URL("http://127.0.0.1:4723"), options);

iOS Options (XCUITest)

Appium 2.x

Setting desired capabilities for iOS testing using the XCUITest driver.

Setting iOS Options
// Requires: import io.appium.java_client.ios.options.XCUITestOptions;
XCUITestOptions options = new XCUITestOptions();
options.setDeviceName("iPhone 14")
    .setPlatformName("iOS")
    .setAutomationName("XCUITest")
    .setPlatformVersion("16.4")
    .setApp("path/to/my.app");

// Create the driver with the options
IOSDriver driver = new IOSDriver(new URL("http://127.0.0.1:4723"), options);

Mobile Locators and API

Finding elements using mobile-specific strategies.

Mobile Locators

Appium 2.x

Appium extends Selenium's `By` class with specialized locators for mobile apps.

Common Mobile Locators
// By accessibility ID (for both Android and iOS)
WebElement element = driver.findElement(AppiumBy.accessibilityId("my_button"));

// By Android UI Automator
WebElement androidElement = driver.findElement(AppiumBy.androidUIAutomator("new UiSelector().text(\"Add\")"));

// By iOS predicate string
WebElement iosElement = driver.findElement(AppiumBy.iOSClassChain("**/XCUIElementTypeButton[`name == 'Add'`]"));

// By Class Name (element type)
WebElement textView = driver.findElement(AppiumBy.className("android.widget.TextView"));

Mobile Gestures

Interacting with the screen through taps, swipes, and scrolls.

Pointer Input API

Appium 2.x

The modern W3C-compliant way to create complex gestures.

Simple Tap Gesture
// Requires: import org.openqa.selenium.interactions.PointerInput;
PointerInput finger = new PointerInput(PointerInput.Kind.TOUCH, "finger");
Point tapPoint = new Point(100, 200);

Sequence tap = new Sequence(finger, 1);
tap.addAction(finger.createPointerMove(Duration.ofMillis(0), PointerInput.Origin.viewport(), tapPoint.x, tapPoint.y));
tap.addAction(finger.createPointerDown(PointerInput.MouseButton.LEFT.as
));
tap.addAction(finger.createPointerUp(PointerInput.MouseButton.LEFT.as
));

driver.perform(Arrays.asList(tap));

Scrolling to an Element

Appium 2.x

A simple way to scroll to an element on the screen.

Scroll to Element (Android)
// Example of scrolling to an element on Android using UI Automator
WebElement myElement = driver.findElement(AppiumBy.androidUIAutomator(
    "new UiScrollable(new UiSelector().scrollable(true))" +
    ".scrollIntoView(new UiSelector().text(\"Element Text\"))"));
myElement.click();

Waits and Synchronization

Ensuring elements are ready before interacting with them.

Explicit Waits

Appium 2.x

Just like in Selenium, explicit waits are a best practice for mobile.

Waiting for an Element
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(15));
WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(AppiumBy.accessibilityId("my_button")));
element.click();

// Waiting for a specific text to appear
wait.until(ExpectedConditions.textToBe(AppiumBy.id("status_message"), "Login Successful"));
