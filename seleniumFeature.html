Core API: Locators and Interactions

Finding elements and performing basic actions on the web page.

Locators

Selenium 4+

Locators are the foundation of Selenium automation. They define strategies to find web elements on a page. Choosing the right locator strategy is crucial for creating reliable, maintainable tests.

Locator Priority & Best Practices
// LOCATOR PRIORITY (Most to Least Reliable):
// 1. ID - Fastest and most reliable (if unique)
// 2. Name - Good for form elements
// 3. CSS Selector - Powerful and flexible
// 4. XPath - Most powerful but slowest
// 5. LinkText/PartialLinkText - For links only
// 6. ClassName - Often not unique
// 7. TagName - Too generic, rarely used

// Example HTML for demonstration:
// <input id="username" name="user" class="form-input" type="text" data-testid="username-field">
// <button class="btn btn-primary" data-qa="login-btn">Login</button>
// <a href="/dashboard" class="nav-link">Go to Dashboard</a>
All Locator Types with Examples
// 1. BY ID - Most preferred (fast and unique)
WebElement usernameInput = driver.findElement(By.id("username"));

// 2. BY NAME - Good for form elements
WebElement userField = driver.findElement(By.name("user"));

// 3. BY CSS SELECTOR - Powerful and flexible
WebElement loginButton = driver.findElement(By.cssSelector("button.btn-primary"));
WebElement inputByAttribute = driver.findElement(By.cssSelector("input[data-testid='username-field']"));
WebElement nestedElement = driver.findElement(By.cssSelector("form .form-input"));

// 4. BY XPATH - Most powerful but slower
WebElement titleByText = driver.findElement(By.xpath("//h1[text()='Welcome']"));
WebElement buttonByAttribute = driver.findElement(By.xpath("//button[@data-qa='login-btn']"));
WebElement followingSibling = driver.findElement(By.xpath("//label[text()='Password']/following-sibling::input"));

// 5. BY LINK TEXT - Exact match for link text
WebElement dashboardLink = driver.findElement(By.linkText("Go to Dashboard"));

// 6. BY PARTIAL LINK TEXT - Partial match for link text
WebElement partialLink = driver.findElement(By.partialLinkText("Dashboard"));

// 7. BY CLASS NAME - Single class only
WebElement navLink = driver.findElement(By.className("nav-link"));

// 8. BY TAG NAME - Usually for finding multiple elements
List<WebElement> allLinks = driver.findElements(By.tagName("a"));
List<WebElement> allInputs = driver.findElements(By.tagName("input"));
Advanced CSS Selector Patterns
// Attribute selectors
driver.findElement(By.cssSelector("input[type='email']"));              // Exact match
driver.findElement(By.cssSelector("input[placeholder*='Enter']"));      // Contains
driver.findElement(By.cssSelector("input[id^='user']"));                // Starts with
driver.findElement(By.cssSelector("input[id$='name']"));                // Ends with

// Pseudo-selectors
driver.findElement(By.cssSelector("tr:nth-child(2)"));                  // Second row
driver.findElement(By.cssSelector("tr:nth-last-child(1)"));             // Last row
driver.findElement(By.cssSelector("input:not([disabled])"));            // Not disabled
driver.findElement(By.cssSelector("option:checked"));                   // Selected option

// Combinators
driver.findElement(By.cssSelector("div > button"));                     // Direct child
driver.findElement(By.cssSelector("form button"));                      // Descendant
driver.findElement(By.cssSelector("label + input"));                    // Adjacent sibling
driver.findElement(By.cssSelector("h2 ~ p"));                          // General sibling

// Complex selectors
driver.findElement(By.cssSelector("table tr:nth-child(odd) td:first-child"));
driver.findElement(By.cssSelector("form input[required]:not([readonly])"));
Advanced XPath Patterns
// Text-based selection
driver.findElement(By.xpath("//button[text()='Submit']"));              // Exact text
driver.findElement(By.xpath("//button[contains(text(),'Submit')]"));    // Contains text
driver.findElement(By.xpath("//button[normalize-space()='Submit']"));   // Normalized text

// Attribute conditions
driver.findElement(By.xpath("//input[@type='text' and @required]"));    // Multiple conditions
driver.findElement(By.xpath("//div[@class and contains(@class,'error')]")); // Has class attribute

// Axes navigation
driver.findElement(By.xpath("//label[text()='Email']/following-sibling::input"));    // Next sibling
driver.findElement(By.xpath("//input[@id='username']/ancestor::form"));              // Ancestor
driver.findElement(By.xpath("//tr[1]/descendant::button"));                          // Descendant
driver.findElement(By.xpath("//button/preceding-sibling::input[1]"));                // Previous sibling

// Position-based selection
driver.findElement(By.xpath("(//button)[1]"));                          // First button
driver.findElement(By.xpath("(//button)[last()]"));                     // Last button
driver.findElement(By.xpath("//tr[position() > 1]"));                   // All except first row

// Dynamic elements
driver.findElement(By.xpath("//div[starts-with(@id,'dynamic-')]"));     // Dynamic ID
driver.findElement(By.xpath("//span[string-length(text()) > 10]"));     // Text length condition
Robust Locator Strategies
// Page Object Model with fallback locators
public class LoginPage {
    private WebDriver driver;
    
    // Primary and fallback locators
    private By usernameField = By.id("username");
    private By usernameFallback = By.name("user");
    private By usernameLastResort = By.cssSelector("input[type='text']:first-of-type");
    
    public WebElement getUsernameField() {
        try {
            return driver.findElement(usernameField);
        } catch (NoSuchElementException e1) {
            try {
                return driver.findElement(usernameFallback);
            } catch (NoSuchElementException e2) {
                return driver.findElement(usernameLastResort);
            }
        }
    }
    
    // Using data attributes for stable locators
    public WebElement getLoginButton() {
        // Prefer data attributes for test automation
        return driver.findElement(By.cssSelector("[data-testid='login-button']"));
    }
    
    // Dynamic element handling
    public WebElement getErrorMessage() {
        // Handle dynamic error messages
        return driver.findElement(By.xpath("//div[contains(@class,'error') and not(contains(@class,'hidden'))]"));
    }
    
    // Table cell selection
    public WebElement getTableCell(int row, int column) {
        String xpath = String.format("//table//tr[%d]//td[%d]", row, column);
        return driver.findElement(By.xpath(xpath));
    }
    
    // Finding by visible text in any element
    public WebElement findByVisibleText(String text) {
        String xpath = String.format("//*[normalize-space(text())='%s' and not(ancestor::*[contains(@style,'display:none') or contains(@style,'visibility:hidden')])]", text);
        return driver.findElement(By.xpath(xpath));
    }
}
Locator Best Practices:
  • Prefer ID and Name: Fastest performance and most reliable
  • Use Data Attributes: data-testid, data-qa for test-specific identification
  • Avoid Brittle Selectors: Don't use absolute XPath or position-dependent selectors
  • CSS over XPath: CSS selectors are generally faster than XPath
Common Pitfalls: Using auto-generated IDs, relying on text that changes, overly complex selectors, and not handling dynamic content.

public boolean isElementReady(WebElement element) {
    try {
        return element.isDisplayed() && element.isEnabled();
    } catch (NoSuchElementException | StaleElementReferenceException e) {
        return false;
    }
}

// CONDITIONAL INTERACTIONS

} else {
    throw new IllegalStateException("Submit button is disabled");
}

Robust Element Interaction Patterns
// SAFE CLICK WITH RETRY
public void safeClick(WebElement element, int maxRetries) {
    for (int i = 0; i < maxRetries; i++) {
        try {
            if (element.isDisplayed() && element.isEnabled()) {
                element.click();
                return;
            }
        } catch (ElementClickInterceptedException e) {
            // Scroll element into view and retry
            ((JavascriptExecutor) driver).executeScript("arguments[0].scrollIntoView(true);", element);
            try { Thread.sleep(500); } catch (InterruptedException ie) {}
        } catch (StaleElementReferenceException e) {
            // Re-find element and retry
            element = driver.findElement(getLocatorFor(element));
        }
    }
    throw new RuntimeException("Failed to click element after " + maxRetries + " attempts");
}

// SMART TEXT INPUT WITH VALIDATION
public void enterText(WebElement inputField, String text) {
    inputField.clear();
    inputField.sendKeys(text);
    
    // Verify text was entered correctly
    String actualValue = inputField.getAttribute("value");
    if (!text.equals(actualValue)) {
        // Retry with character-by-character input
        inputField.clear();
        for (char c : text.toCharArray()) {
            inputField.sendKeys(String.valueOf(c));
            try { Thread.sleep(50); } catch (InterruptedException e) {}
        }
    }
}

// DROPDOWN SELECTION HELPER
public void selectDropdownOption(WebElement dropdown, String optionText) {
    Select select = new Select(dropdown);
    
    // Try different selection methods
    try {
        select.selectByVisibleText(optionText);
    } catch (NoSuchElementException e) {
        try {
            select.selectByValue(optionText);
        } catch (NoSuchElementException e2) {
            // Find option by partial text match
            List<WebElement> options = select.getOptions();
            for (WebElement option : options) {
                if (option.getText().contains(optionText)) {
                    option.click();
                    return;
                }
            }
            throw new NoSuchElementException("Option not found: " + optionText);
        }
    }
}

// CHECKBOX/RADIO BUTTON STATE MANAGEMENT
public void setCheckboxState(WebElement checkbox, boolean shouldBeChecked) {
    boolean isCurrentlyChecked = checkbox.isSelected();
    if (isCurrentlyChecked != shouldBeChecked) {
        checkbox.click();
    }
}

// FORM FILLING HELPER
public void fillForm(Map<String, String> formData) {
    for (Map.Entry<String, String> entry : formData.entrySet()) {
        WebElement field = driver.findElement(By.name(entry.getKey()));
        
        if (field.getTagName().equalsIgnoreCase("select")) {
            selectDropdownOption(field, entry.getValue());
        } else if (field.getAttribute("type").equals("checkbox")) {
            setCheckboxState(field, Boolean.parseBoolean(entry.getValue()));
        } else {
            enterText(field, entry.getValue());
        }
    }
}
Interaction Best Practices:
  • Always Check State: Verify element is displayed and enabled before interaction
  • Handle Exceptions: Catch StaleElementReferenceException and ElementClickInterceptedException
  • Validate Results: Confirm actions had the expected effect
  • Use Appropriate Methods: Select class for dropdowns, Actions for complex interactions
Common Issues: Element not in viewport, overlapping elements, dynamic content loading, and stale element references.

Advanced Interactions

Complex user actions like drag-and-drop, hover, and keyboard inputs.

Actions API

Selenium 4+

Simulate complex user interactions with a fluent API.

Drag and Drop Example
Actions actions = new Actions(driver);
WebElement draggable = driver.findElement(By.id("draggable"));
WebElement droppable = driver.findElement(By.id("droppable"));

actions.dragAndDrop(draggable, droppable).perform();

// Chaining multiple actions
WebElement elementToHover = driver.findElement(By.id("menu-item"));
actions.moveToElement(elementToHover)
       .click()
       .sendKeys(Keys.ENTER)
       .perform();

Relative Locators

Selenium 4+

Find elements based on their position relative to another element on the page.

Relative Locators Example
// Find the element with the tag 'a' that is to the right of the login button.
WebElement link = driver.findElement(with(By.tagName("a")).toRightOf(By.id("login-btn")));

// Find the element with the tag 'input' that is above the password input.
WebElement usernameField = driver.findElement(with(By.tagName("input")).above(By.id("password-field")));

// More examples
WebElement passwordField = driver.findElement(with(By.id("password-field")).below(usernameField));
WebElement header = driver.findElement(with(By.tagName("h1")).near(By.id("main-div")));

Waits and Synchronization

Handle dynamic content and asynchronous loading gracefully.

Fluent and Explicit Waits

Selenium 4+

Wait for a specific condition to be met before continuing execution.

Implicit Wait
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
// This applies globally and can be inefficient.
Explicit Wait
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("dynamic-element")));

// Custom wait conditions
Wait<WebDriver> fluentWait = new FluentWait<WebDriver>(driver)
    .withTimeout(Duration.ofSeconds(30))
    .pollingEvery(Duration.ofSeconds(5))
    .ignoring(NoSuchElementException.class);

WebElement foo = fluentWait.until(driver -> {
    return driver.findElement(By.id("foo"));
});
