TestNG Annotations

The core building blocks for structuring your test code.

Core Annotations

TestNG

Annotations for test methods and lifecycle hooks.

Test Class Example
import org.testng.annotations.Test;
import org.testng.annotations.BeforeMethod;
import org.testng.annotations.AfterMethod;

public class MyTests {

    @BeforeMethod
    public void setup() {
        System.out.println("This runs before each test method.");
    }

    @Test
    public void testLoginFunctionality() {
        System.out.println("Executing testLoginFunctionality.");
        // Test logic here
    }

    @Test
    public void testSearchFunctionality() {
        System.out.println("Executing testSearchFunctionality.");
        // Test logic here
    }

    @AfterMethod
    public void teardown() {
        System.out.println("This runs after each test method.");
    }
}

Suite & Class Annotations

TestNG

Used for setup and teardown at a higher level (suite or class).

Suite/Class Level Example
import org.testng.annotations.*;

public class SuiteTest {

    @BeforeSuite
    public void beforeSuite() {
        System.out.println("Setting up before the entire test suite.");
    }

    @BeforeClass
    public void beforeClass() {
        System.out.println("Setting up before any methods in this class.");
    }

    @Test
    public void firstTest() {
        System.out.println("Running firstTest...");
    }

    @Test
    public void secondTest() {
        System.out.println("Running secondTest...");
    }

    @AfterClass
    public void afterClass() {
        System.out.println("Cleaning up after all methods in this class.");
    }

    @AfterSuite
    public void afterSuite() {
        System.out.println("Cleaning up after the entire test suite.");
    }
}

Test Suite XML

A powerful way to group and configure your test runs.

Creating a Test Suite

TestNG

Define which classes and methods to run, and in what order, using a `testng.xml` file.

Example `testng.xml`
<!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd" >
<suite name="My Automation Suite" verbose="1" >
  <test name="My Regression Tests" >
    <classes>
      <class name="com.example.MyTests" />
      <class name="com.example.SuiteTest" />
    </classes>
  </test>
</suite>

Parameters and Data-Driven Testing

Run the same test with different sets of data.

XML Parameters

TestNG

Passing values to test methods directly from the `testng.xml` file.

Test Class (no parameters)
// No way to pass dynamic data
@Test
public void loginTest() {
    String username = "user1";
    String password = "password1";
    // ... test logic
}
Test Class with @Parameters
// In the test class
import org.testng.annotations.Parameters;

public class ParameterTest {
    @Test
    @Parameters({"username", "password"})
    public void loginTest(String username, String password) {
        System.out.println("Username is: " + username);
        System.out.println("Password is: " + password);
    }
}

// In testng.xml
<suite name="Param Suite">
    <test name="Login Test">
        <parameter name="username" value="admin"/>
        <parameter name="password" value="secret"/>
        <classes>
            <class name="com.example.ParameterTest"/>
        </classes>
    </test>
</suite>

Data Provider

TestNG

More flexible data-driven testing using a Java method to provide test data.

Data Provider Example
import org.testng.annotations.DataProvider;
import org.testng.annotations.Test;

public class DataProviderTest {
    @DataProvider(name = "loginData")
    public Object[][] provideData() {
        return new Object[][] {
            {"user1", "pass1"},
            {"user2", "pass2"}
        };
    }

    @Test(dataProvider = "loginData")
    public void loginTest(String username, String password) {
        System.out.println("Testing with: " + username + " and " + password);
    }
}

Parallel Execution

Run tests concurrently to speed up your automation suite.

Configuring Parallelism

TestNG

Use the `testng.xml` file to define how many threads and which test components should run in parallel.

Parallel Execution in `testng.xml`
<!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd" >
<suite name="Parallel Suite" parallel="tests" thread-count="2">
  <test name="Browser Test - Chrome">
    <classes>
      <class name="com.example.BrowserTests" />
    </classes>
  </test>
  <test name="Browser Test - Firefox">
    <classes>
      <class name="com.example.BrowserTests" />
    </classes>
  </test>
</suite>
